### pa2kva

In SV32, the 32-bit VA space is split across 1024 megapages (each 4MB). This code uses two of them:
- VA 0x00000000  [l1pt[0]]           → user space (lowest megapage)
- VA 0xFFC00000  [l1pt[1023]]        → kernel space (uppermost megapage)

kenrel megapage mapping: 0xFFC00000 gets mapped to DRAM_BASE with `l1pt[PTES_PER_PT-1] = (DRAM_BASE/RISCV_PGSIZE << PTE_PPN_SHIFT) | ...`

so, pa2kva: 
1. KVA = PA - DRAM_BASE + 0xFFC00000
2. KVA = PA - 0x80000000 + 0xFFC00000
3. KVA = PA - 0x80000000 - 0x00400000 because 0xFFC00000 = -0x400000 in 32-bit
4. KVA = PA - DRAM_BASE - MEGAPAGE_SIZE

thus, pa -> kva intentionally wraps aroudn in 32b
