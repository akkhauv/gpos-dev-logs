```
typedef struct
{
  long gpr[32];
  long sr;
  long epc;
  long badvaddr;
  long cause;

```
- gpr: all general-purpose registers
- epc: exception program counter -- where to resume execution
- status register: defines CPU mode (user vs supervisor)

```
tf.epc = test_addr - DRAM_BASE;
```
- physical RAM starts at DRAM_BASE
- user's program is complied to run, starting at address 0
- thus -- calculating the test's address - DRAM base, we get the virthal address of the test start
- this is because we've mapped user to the lowermost megapage at 0x0: 

```
Address: 0x00001000
Binary:  0000 0000 0000 0000 0001 0000 0000 0000
         |        | |      |
Bits:    31      22 21    12
         [ 1stlvl ] [ 2ndlvl ] [  Offset  ]
```
- because of this mapping, we get indexed into the pagetable
- it knows the 1st lvl page table is 0, so we go to index 0 of the l1 page taable
- since this has already been mapped to the user space with `l1pt[0] = ((pte_t)user_l2pt >> PGSHIFT << PTE_PPN_SHIFT) | PTE_V;`, we go to the l2 page table for user 
- nothing is here yet! a page fault handler runs

### handle_fault

```
addr = addr/PGSIDZE * PGSIZE
```
- rounds down the address to the page that caused it 

```
if (user_llpt[addr/PGSIZE]) {
  if (!(user_llpt[addr/PGSIZE] & PTE_A)) {
    user_llpt[addr/PGSIZE] |= PTE_A;
  } else {
    assert(!(user_llpt[addr/PGSIZE] & PTE_D) && cause == CAUSE_STORE_PAGE_FAULT);
    user_llpt[addr/PGSIZE] |= PTE_D;
  }
  flush_page(addr);
  return;
}
```
- if the user's page table was already loaded in, then we detour to this
- IMPORTANT: strickly a user-space page fault handler
- the kernel doesn't fault because it doesn't do two-level paging; instead, the MMU just returns the directly translated address
  - this is a direct-mapped megapage: it's also known as identity-offset mapping
