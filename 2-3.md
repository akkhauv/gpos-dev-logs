### multicore and MMU 

- page table modifications -- rare
- TLB coherence is enforced i software (unlike other stuff)
- TLB shootdown -- hwne i go to modify the page table, i send interrupts to all the other cores, and force everyone to enter the OS kernel and collectively kill the relevant pages in my TLB that are to be modified, and then the offending core will modify the TLB and u return to program execution
- filtering mechanism -- maybe the OS knows which cores will have a cached copy somewhere and only interrupt those guys
- basic idea -- everyone stops, they go nuke the offending entries, do the modification, and then return to the dfprogram
- un need this whenever u modify the page table
- sfence.vma: synchronizes TLB with the page table (invalidates everything lol)
